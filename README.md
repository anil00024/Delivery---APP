# STRYDE — India Sneaker Store (React + Vite + Tailwind)

A full rebuild of the original SneakStore demo into a distinct, India-focused sneaker
marketplace app called **STRYDE** — new brand identity, new copy, INR pricing with a
real Indian product catalog, and a stack of new features across every screen.

## Run it

```bash
npm install
npm run dev
```

Then open the printed local URL (usually `http://localhost:5173`). To produce a static
production build:

```bash
npm run build
npm run preview
```

The whole app lives in `src/App.jsx` (single-file, ~1,860 lines) plus `src/index.css`
for global styles and `tailwind.config.js` for the design tokens.

## What changed from the original

**Brand & visual identity**
- New name/wordmark **STRYDE**, dark "ink" + "marigold" palette instead of the old
  emerald theme, `Bebas Neue` display type + `Inter` body type + `JetBrains Mono` for
  prices/order IDs, loaded via Google Fonts in `index.html`.
- A recurring "shoebox stamp" tag component (`StampTag`) used for New Drop / Bestseller
  / Sale badges instead of plain pill labels.
- A brand-name marquee ticker and rotating hero banner carousel on the home screen.
- Category-aware sneaker illustrations: the procedural `ShoeIcon` now renders a
  different silhouette (tread lines for trail, speed stripe for running) per category.
- Full dark mode toggle (Profile → Dark mode), applied across shell, nav, cards and
  every screen's shared components.

**Catalog**
- 12 real Indian-market products (Campus, Sparx, Bata, Liberty, Red Tape, ASIAN, Puma,
  Adidas, Nike, Woodland, Skechers, Bacca Bucci) with INR pricing, ratings, descriptions,
  feature bullet points, and 2–3 colorway swatches each.
- Prices, discounts, cart, checkout and orders all use ₹ / `en-IN` formatting.

**New features / sections**
- **Search overlay** with recent searches and trending search suggestions.
- **Sort & Filter bottom sheet** — sort by popularity/price/rating, price slider, brand
  checkboxes, with an active-filter counter badge.
- **Trending** and **New Drops** carousels on Home.
- **Product detail**: colorway swatches that swap the illustration, a Details/Reviews
  tab switch, a pincode delivery-estimate checker, and a review submission form that
  appends to a live reviews list.
- **Cart**: save-for-later, gift wrap toggle, wallet-points redemption toggle, promo
  codes, full price breakdown.
- **Address book**: multiple saved addresses, add/delete/set default, used at checkout.
- **Checkout**: address selector, 4 payment methods (UPI/Card/COD/Wallet).
- **Wallet & rewards** screen with points balance and transaction history; every order
  earns 2% back in points.
- **Help & support** screen with an FAQ accordion and chat/call/email entry points.
- **Notifications**: "mark all as read" action.
- Phone-number/OTP sign-up path alongside the original email/password flow, plus a
  4-slide onboarding flow with India-specific copy (pan-India delivery, 7-day returns).

## v2.1 update — fixes & new features

**Bugs fixed**
- **Price now reacts to size.** Extended sizes (UK 6, 7, 10, 11) carry a real surcharge
  (`sizeSurcharge()` / `priceForSize()` in `App.jsx`), shown on every size button, the
  product price, and carried correctly through cart → checkout → order total. Previously
  the price never changed when you picked a different size — that's fixed.
- **Login/signup buttons that silently did nothing.** The old "Forgot password" button
  used a native `alert()`, which several embedded/mobile webviews block or swallow
  silently — it looked like a dead button. Replaced with an in-app toast. Errors are
  now cleared when you switch between Login/Signup or Email/Phone, and there's a
  one-tap **"Use a demo account"** link on the auth screen if you ever get stuck.
  Phone sign-in now has a working **Send OTP** button (use `1234` in this demo).

**Real product images**
- Every product now has a real raster photo (`public/product-images/<id>.jpg`) shown
  via an `<img>` tag with automatic fallback to the old icon if it ever fails to load.
  These are studio-style renders generated by `scripts/gen_images.py` (gradient
  backdrop, soft shadow, shaded silhouette) rather than hotlinked photos — hotlinking
  real branded product photography from the web is unreliable (links rot, get
  rate-limited) and legally murky to redistribute in a codebase. **To use real
  photography**, just drop JPGs into `public/product-images/` named `p1.jpg` … `p12.jpg`
  (or update the `image` field in `PRODUCTS`) — the app will pick them up automatically.

**Checkout, rebuilt**
- Checkout is now a proper 3-step flow — **Address → Payment → Review** — with a
  progress stepper, back-navigation between steps, and real per-method validation:
  UPI ID format check, 16-digit card number/expiry/CVV checks, a COD order-value cap
  (₹5,000), and a wallet-balance check. Step 1 also has an optional delivery-instructions
  field, and Step 3 requires accepting the return policy before the order can be placed.

**Chatbot**
- A floating STRYDE Assistant button now sits above the bottom nav on every main
  screen. It's a rule-based chat widget (`ChatBot` component + `botReply()`) that
  answers questions about order tracking, returns, payments, sizing and delivery, with
  quick-reply chips. "Live chat" in Help & Support and "Need help with this order?" on
  the order detail screen both open the same assistant.

**Works properly on real phones**
- The app shell now fills the actual viewport (`100dvh`, safe-area insets for notches
  and home indicators) on phone-width screens instead of showing the decorative phone
  bezel — the bezel/mock-phone frame only appears as a preview on wider (desktop)
  screens. Inputs are locked to 16px to stop iOS Safari's auto-zoom-on-focus, and
  `viewport-fit=cover` / mobile-web-app meta tags are set in `index.html`.

## v2.2 update — account bug fix, image galleries, feature pass

**Critical fix: fresh accounts no longer inherit demo data**
- Previously every session started with a pre-filled cart, wishlist, wallet balance
  and order history — so a brand-new signup or login looked like it already had
  orders. Login and Signup now both call `startFreshAccount()`, which gives a genuinely
  empty cart, empty wishlist, empty order history, no saved addresses, and a 50-point
  welcome bonus. Guest browsing does the same. The populated example data (orders,
  wallet balance, wishlist, saved addresses) is now only loaded by the explicit
  **"Use a demo account"** link on the auth screen (`startDemoAccount()`), so you can
  still see every screen fully stocked on demand without it masquerading as your own
  history.

**Every product now has a 3-shot image gallery**
- Each product ships with `<id>-1.jpg` (hero), `<id>-2.jpg` (angled view), `<id>-3.jpg`
  (sole close-up) in `public/product-images/`, generated by the updated
  `scripts/gen_images.py`. The product page has a real swipeable/tappable gallery with
  thumbnails, arrow controls, and dot indicators.

**New features across the app**
- **Product page**: quantity stepper before adding to cart, a "You may also like"
  related-products carousel (same category), and a share button.
- **Home**: a live "Deal of the day" countdown, a "Recently viewed" carousel that
  tracks what you've opened, and a quick-add (+) button on every product card that
  adds the default size straight to your cart without opening the product.
- **Orders**: Reorder (re-adds every item from a past order to your cart) on both the
  orders list and order detail, plus a Cancel order flow (with confirmation) for
  orders still in Processing.
- **Wishlist**: "Move all to cart" bulk action.
- **Notifications**: a preferences panel to toggle order-update vs promotional alerts.
- **Profile & Wallet**: a Refer & Earn card with a generated referral code (copy to
  share), and a "Redeem 100 points for a ₹100 coupon" action on the Wallet screen.
- **Help & Support**: a search box that filters the FAQ list live.

## Notes

- Product photography is procedurally generated (see above) so the app has zero
  external dependencies for images and works fully offline after `npm install`.
- State is in-memory only (React state), so refreshing the page resets cart/orders —
  swap in `localStorage`/an API layer if you want persistence.
